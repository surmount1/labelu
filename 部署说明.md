1、k8s 部署的时候：需要把.env 中的环境变量加入到deployments的yaml 中的env 中

2、ENV XDG_DATA_HOME=/data 逻辑是这样的：
    项目调用了 appdirs 的 user_data_dir，应用名是 labelu
    在 Linux 环境里，appdirs 会参考 XDG_DATA_HOME 这个环境变量
    你在镜像里把 XDG_DATA_HOME 设成了 /data
    所以 user_data_dir("labelu") 最终就会变成 /data/labelu
    也就是说，这不是直接把数据目录写死成 /data/labelu，而是通过 appdirs 的规则“算出来”的。

    这套关系可以理解成：

    XDG_DATA_HOME = /data
    appdirs 看到应用名是 labelu
    最终 BASE_DATA_DIR = /data/labelu
    MEDIA_ROOT = /data/labelu/media

6、docker 部署：
启动：
LABELU_DATA_DIR=/data/labelu docker compose up -d
停止：docker compose down
查看日志：
docker compose logs -f

7、docker compose 中的volume 的意思是：
    现象：
        volumes:
        - ${LABELU_DATA_DIR:-./.labelu-data}:/data

    原因：
        如果你在启动时传了 LABELU_DATA_DIR
        那就用你传入的那个目录

        如果你没传
        那就默认使用当前项目目录下的 ./.labelu-data

        所以它不是“不直接用 LABELU_DATA_DIR”，而是“优先用 LABELU_DATA_DIR，没有就回退到默认目录”。
