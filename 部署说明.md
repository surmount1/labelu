# 部署说明

## 1. K8s 部署

需要把 .env 中的环境变量加入 deployments 的 yaml 配置里的 env 段。

## 2. XDG_DATA_HOME=/data 的作用

项目调用了 appdirs 的 user_data_dir，应用名是 labelu。

在 Linux 环境里，appdirs 会参考 XDG_DATA_HOME 这个环境变量。你在镜像里把 XDG_DATA_HOME 设成了 /data，所以 user_data_dir("labelu") 最终就会变成 /data/labelu。

也就是说，这不是直接把数据目录写死成 /data/labelu，而是通过 appdirs 的规则计算出来的。

关系如下：

```text
XDG_DATA_HOME = /data
appdirs 看到应用名是 labelu
最终 BASE_DATA_DIR = /data/labelu
MEDIA_ROOT = /data/labelu/media
```

## 4. Docker 构建

Docker build 时需要区分平台，尤其是 macOS 构建 Linux 镜像时。示例：

```bash
docker buildx build --platform linux/amd64 -t harbor.ruijie.com.cn/ai/labelu:1.3.2-fix7 --push .
```

## 5. 开发环境启动

```bash
uv run uvicorn labelu.main:app --host 0.0.0.0 --port 8000
```

## 6. Docker 部署

启动：

```bash
LABELU_DATA_DIR=/data/labelu docker compose up -d
```

停止：

```bash
docker compose down
```

查看日志：

```bash
docker compose logs -f
```

## 7. Docker Compose 中 volume 的含义

配置示例：

```yaml
volumes:
  - ${LABELU_DATA_DIR:-./.labelu-data}:/data
```

含义：

- 如果启动时传了 LABELU_DATA_DIR，就使用传入的目录。
- 如果没有传，就默认使用当前项目目录下的 ./.labelu-data。

所以它不是“不直接用 LABELU_DATA_DIR”，而是“优先用 LABELU_DATA_DIR，没有就回退到默认目录”。

## 8. Git 拉代码后初始化环境

```bash
git clone 你的仓库地址
cd labelU
brew install uv
uv venv --python 3.11
source .venv/bin/activate
uv sync
cp .env.example .env
openssl rand -hex 32
uv run uvicorn labelu.main:app --host 0.0.0.0 --port 8000
```

说明：

- uv sync：根据项目依赖声明和锁文件安装依赖。
- cp .env.example .env：复制环境变量模板文件。
- openssl rand -hex 32：生成随机密钥，可填入 PASSWORD_SECRET_KEY。
